# Media Forge 使用说明

Media Forge 是一个全栈媒体处理应用，前端提供初始化配置、登录、仪表盘、爬虫任务、影片内容管理和存储任务管理等页面。

本文主要介绍各页面的操作功能。开发和模块结构说明见 [frontend/README.md](frontend/README.md)。

## 快速入口

开发环境常用地址：

- 前端开发服务：`http://localhost:18643`
- 后端服务：`http://localhost:18642`

首次启动时，系统会进入 `/init` 完成 PostgreSQL 和 Redis 配置；初始化完成后进入 `/login` 登录。

## 全局布局

登录后页面使用统一控制台布局：

- 左侧菜单：进入仪表盘、爬虫、存储、内容管理等模块。
- 顶部标题：显示当前页面名称。
- 侧边栏按钮：展开或收起左侧菜单。
- 主题按钮：切换明亮/暗色主题。
- 用户区：显示当前用户，点击退出按钮可退出登录。
- 标签页栏：记录已访问页面，可切换、关闭和通过右键菜单管理页面缓存。

## 初始化配置 `/init`

用于首次运行时配置后端依赖服务。

可操作功能：

- 填写 PostgreSQL 主机、端口、数据库名、用户名、密码、连接超时、连接池和重试参数。
- 点击“测试 PostgreSQL 连接”验证数据库连接。
- 填写 Redis 主机、端口、密码、超时和最大连接数。
- 点击“测试 Redis 连接”验证 Redis 连接。
- 点击“保存并进入登录页”保存配置并跳转登录。

如果数据库和 Redis 已配置完成，访问该页面会自动跳转到登录页。

## 登录页 `/login`

用于进入系统控制台。

可操作功能：

- 输入账号和密码登录。
- 默认表单值为 `admin/admin123`。
- 勾选“记住密码”。
- 通过右上角主题按钮切换明亮/暗色主题。
- 登录成功后进入原目标页面或仪表盘。

## 仪表盘 `/`

用于查看系统运行概览。

可操作功能：

- 查看系统状态和最后刷新时间。
- 点击刷新按钮重新拉取首页数据。
- 查看采集队列、任务配置、影片库、存储索引等关键指标。
- 查看采集运行状态分布和趋势图。
- 在“最近工作”中切换“最近采集运行”和“最近存储任务”。
- 点击最近采集运行进入运行详情。
- 点击最近存储任务进入存储任务详情。
- 查看告警和部分模块降级提示。

## 爬虫任务列表 `/crawler/tasks`

用于管理 JavDB 采集任务。

可操作功能：

- 查看任务统计：总数、空闲中、运行中、排队中、停止中。
- 查看每个任务的网盘路径、URL 名称、最后爬取时间和运行状态。
- 点击“新建任务”进入任务创建页。
- 点击“临时任务”创建一次性详情页采集任务。
- 使用任务卡片上的启用开关启用或禁用任务。
- 点击“爬取”选择“增量爬取”或“全量爬取”。
- 点击“URL 爬取”选择任务下的部分 URL 单独执行采集。
- 对运行中或排队中的任务点击“停止”。
- 对已停止任务点击“重启”。
- 对空闲任务点击编辑或删除。
- 页面会通过实时事件更新任务运行状态。

### 临时任务弹窗

用于对指定任务提交一次性的 JavDB 详情页 URL。

可操作功能：

- 选择归属任务。
- 添加、删除详情页 URL。
- 最多提交 50 条详情页 URL。
- 点击“创建临时任务”提交。
- 任务列表加载失败时可点击“重新加载任务”。

### URL 爬取弹窗

用于只采集任务中的部分 URL。

可操作功能：

- 多选任务 URL。
- 选择“增量爬取”或“全量爬取”。
- 点击“开始爬取”提交。

## 新建/编辑爬虫任务 `/crawler/tasks/new`、`/crawler/tasks/$id/edit`

用于创建或维护采集任务。

可操作功能：

- 填写任务名称。
- 设置启用状态。
- 填写网盘路径；新建时任务名称可自动同步为路径，编辑时路径不可修改。
- 添加多个来源 URL。
- 在 URL 卡片视图和列表视图之间切换。
- 为每条 URL 配置 URL 类型、URL 名称、是否需要磁链、是否需要中文字幕、排序规则。
- 自动识别 URL 类型。
- 自动提取 URL 名称。
- 预览最终采集 URL。
- 删除多余 URL。
- 点击“创建”或“更新”保存任务。
- 点击“取消”放弃修改并返回任务列表。

保存时会校验 URL 是否重复，以及 URL 类型是否可识别。

## 爬虫运行记录 `/crawler/runs`

用于查看和管理采集运行历史。

可操作功能：

- 查看任务名称、运行状态、运行模式和创建时间。
- 分页浏览运行记录。
- 点击“详情”进入运行详情页。
- 对排队中或运行中的记录点击“停止”。
- 对已停止或失败的记录点击“重启”。
- 对非运行中的记录点击“删除”，只删除运行记录和子任务记录，不删除影片数据。
- 页面会通过实时事件更新运行状态。

## 爬虫运行详情 `/crawler/runs/$id`

用于查看单次采集运行的详情、子任务和日志。

可操作功能：

- 查看运行任务、状态、模式、创建时间、开始时间、完成时间和错误信息。
- 对排队中或运行中的运行点击“停止”。
- 对已停止或失败的运行点击“重启”。
- 查看子任务统计指标。
- 按状态筛选子任务。
- 按关键字搜索子任务。
- 分页查看子任务。
- 对失败子任务执行单条重试。
- 勾选失败子任务后批量重试。
- 一键重试当前列表中的全部失败子任务。
- 查看运行日志时间线。
- 页面会通过实时事件追加日志并刷新运行和子任务状态。

## 爬虫配置 `/crawler/config`

用于调整采集参数和 JavDB Cookie。

可操作功能：

- 配置列表线程数、详情线程数、最大翻页数。
- 配置列表页和详情页请求延迟。
- 配置请求超时和安全验证等待时间。
- 配置增量爬取阈值。
- 点击“保存配置”保存爬取参数。
- 在 Monaco JSON 编辑器中编辑 Cookie 配置。
- 点击“格式化”格式化 Cookie JSON。
- 点击“保存 Cookie”保存 Cookie 配置。
- JSON 格式错误时页面会提示错误位置。

## 影片列表 `/content/movies`

用于浏览、筛选和处理已采集影片。

可操作功能：

- 查看影片番号、标题、评分、发行日期、时长、演员、标签和存储状态。
- 按任务筛选影片。
- 按番号或标题搜索。
- 按演员、标签、导演、片商、系列筛选或排除。
- 按存储状态筛选。
- 按评分、演员数量、发行日期、入库时间筛选。
- 按番号、发行日期、评分、抓取时间排序。
- 点击“搜索”应用筛选。
- 点击“刷新”重置筛选并刷新列表。
- 点击“配置”打开筛选条件配置抽屉。
- 点击“详情”查看影片详情。
- 点击“推送”创建单个影片存储任务。
- 点击“CD2同步”从 CloudDrive2 同步单个影片存储状态。
- 点击“删除”删除单个影片。
- 勾选多条影片后点击“批量推送”。
- 勾选多条影片后点击“批量删除”。
- 点击“存储索引”选择全量或增量刷新存储索引。
- 点击“索引同步”按当前筛选范围同步影片存储状态。
- 页面会通过实时事件更新影片存储状态。

### 影片详情抽屉

可查看和操作：

- 番号、标题、发行日期、时长、评分、导演、制作商、系列、演员、标签。
- 中文字幕、大小、封面、最佳磁力、磁力链接、来源 URL。
- 存储位置和最后同步时间。
- 复制磁力链接和存储路径。
- 点击导演、制作商、系列、演员或标签，快速回填为列表筛选条件。

### 筛选条件配置抽屉

可操作功能：

- 显示或隐藏筛选条件。
- 上移或下移筛选条件显示顺序。
- 为部分筛选条件设置默认值。
- 点击“保存配置”保存筛选栏配置。

### 推送存储弹窗

可操作功能：

- 设置存储任务别名。
- 选择“单个”或“多个”存储模式。
- 单个影片且单个模式下可选择目标文件夹。
- 批量模式下显示已选择影片数量。
- 确认后创建存储推送任务。

## 存储配置 `/storage/config`

用于配置 CloudDrive2 或存储服务连接与任务执行参数。

可操作功能：

- 配置 gRPC 主机地址。
- 填写新的 API Token；留空表示不修改已有 Token。
- 配置请求超时和连接超时。
- 配置下载根目录和目标文件夹。
- 开关“使用任务子文件夹”。
- 开关“自动创建目标文件夹”。
- 配置操作延迟、下载轮询间隔、重试延迟和最大重试次数。
- 配置下载轮询最大次数。
- 配置每个子任务最多尝试磁力条数。
- 配置最小视频大小。
- 配置视频扩展名。
- 点击“测试连接”验证存储服务连接。
- 点击“保存配置”保存配置。
- 点击“重置”重新加载后端保存的配置。

## 存储任务列表 `/storage/tasks`

用于查看和管理影片入库存储任务。

可操作功能：

- 查看任务别名、状态、模式、处理进度、创建时间。
- 查看总数、成功数、失败数、跳过数。
- 点击“刷新”重新加载列表。
- 分页浏览任务。
- 点击“详情”进入存储任务详情。
- 对排队中或运行中的任务点击“停止”。
- 对已停止或失败的任务点击“重启”。
- 对非运行、非停止中的任务点击“删除”；删除主任务、子任务和日志，不删除网盘文件。
- 页面会通过实时事件更新任务状态和进度。

## 存储任务详情 `/storage/tasks/$id`

用于查看单个存储主任务的执行进度和子任务。

可操作功能：

- 查看任务别名、任务编号、状态、模式、创建时间、完成时间。
- 查看任务进度、成功数、失败数、跳过数。
- 查看错误信息。
- 对排队中或运行中的任务点击“停止”。
- 对已停止或失败的任务点击“重启”。
- 查看子任务明细：番号、状态、当前步骤。
- 点击子任务“详情”进入子任务详情页。
- 页面会通过实时事件刷新主任务和子任务状态。

## 存储子任务详情 `/storage/tasks/subtasks/$id`

用于排查单个影片存储子任务。

可查看内容：

- 子任务基本信息。
- 相关文件信息。
- 步骤时间线，包括每个步骤的最新日志。
- 完整任务日志。
- 日志级别和错误信息。
- 页面会通过实时事件更新子任务状态和日志。

## Docker 打包

项目提供 Makefile 打包目标：

```bash
make docker-build-amd64
make docker-build-arm64
```

Docker Hub 访问不稳定时，可以通过基础镜像变量切换镜像源：

```bash
make docker-build-amd64 \
  NODE_BASE_IMAGE=mirror.gcr.io/library/node:22-bookworm-slim \
  PYTHON_BASE_IMAGE=mirror.gcr.io/library/python:3.12-slim
```
